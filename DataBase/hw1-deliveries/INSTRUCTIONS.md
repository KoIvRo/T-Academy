# Инструкция по выполнению домашнего задания

## Общая информация

В этом репозитории для каждого задания подготовлены отдельные файлы, в которых необходимо написать SQL-запросы. Общая
информация о заданиях, а также ссылки на эти файлы содержатся в **README.md**.

Каждое задание соответствует **одному SQL-запросу**, который должен быть реализован в отдельном файле.

## Расположение файлов

Файлы для выполнения заданий находятся в папке:

```
student_solutions/
```

**Важно:** кроме содержания этих файлов ничего трогать **нельзя**. Все остальные скрипты и настройки предназначены только для
проверки и тестирования решений.

Файлы имеют формат:

```
001_all_books.sql
002_unique_last_names.sql
003_authors_list.sql
...
```

где номер соответствует номеру задания, а имя файла — alias задания.

## Пример заполнения файла

Для файла `001_all_books.sql` пример содержания может быть таким:

```sql
-- Запрос: вывести все названия книг
SELECT title
FROM books;
```

Каждый файл должен содержать только **один SQL-запрос**, который выполняет требуемое задание.

## Локальная проверка

Вы можете проверить свои решения локально с помощью Docker. Для этого:

1. Убедитесь, что установлен Docker.
2. Из корня проекта запустите команду:

```bash
docker compose up -d
```

3. Подключитесь к базе, используйте SQL-клиент или запустите **CI чекер** для автоматической проверки решений:

```bash
pip3 install -r scripts/requirements.txt
python3 scripts/ci_checker.py
```

## Ветка и MR

1. Создайте ветку с названием `hw1`:

```bash
git checkout -b hw1
```

2. Сделайте все изменения **только в файлах из папки `student_solutions/`**.
3. После завершения работы откройте Merge Request **из вашей ветки `hw1` в `master` вашего собственного форка**
   репозитория.
   ⚠️ **Не в основной репозиторий**, только в свой форк!

## Рекомендации

* Не меняйте содержимое папки `scripts/` и любые другие системные файлы.
* Каждое задание выполняйте в отдельном файле, чтобы проверка CI проходила корректно.
* Проверяйте решения локально перед отправкой.
* Следите за правильностью типов данных и синтаксисом SQL.
