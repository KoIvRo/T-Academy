# Документация
## Генерация
Для генерации выполнить в подкаталоге src:
```bash
python main.py generate --algorithm=dfs --width=1 --height=1 --output=file.txt --unicode=1 --terrain=1
```
Аргументы `output`, `unicode`, `terrain` являются необязательными
При отсутствии аргумента `output` вывод будет произведен в консоль
Наличие аргумента `unicode` отвечает за отрисовку лабиринта с использованием специальных символов
Наличие аргумента `terrain` отвечает за добавление различных поверхностей:
Ухудшающие поверхности: `&` - песок, `~` - болото. Улучшающие поверхности: `+` - монета
#### Обратите внимание
Реальные размеры лабиринта увеличиваются на 1 клетку в длину и ширину, из за добавления стен
```bash
python main.py generate --algorithm=dfs --width=1 --height=1
```
```text
###
# #
###
```
## Решение
Для решения выполнить в подкаталоге src:
```bash
python main.py solve --algorithm=astar --start=1,1 --end=3,3 --file=file.txt --output=res.txt --unicode=1
```
Аргументы `output` и `unicode` также являются необязательными
На вход из файла указанного в аргументе `file` может поступать лабиринт как в формате unicode, так и в обычном
#### Обратите внимание
Нумерация клеток начинается с левого верхнего угла и заканчивается в правом нижнем.
Первой цифрой указывается координата по x, второй по y
Координаты: `O` = 1,1, `X` = 3,3
```text
#####
#O# #
# # #
#  X#
#####
```

# Функциональные требования
- [x] Реализация 2-x алгоритмов генерации лабиринта методом поиска в глубину(dfs) и алгоритм Прима(prim).
- [x] Реализация 2-х алгоритмов для поиска пути из заданной начальной точки O в конечную точку X: A-star(astar) и Дейкстры(dijkstra).
- [x] Ввод конфигурации лабиринта, начальной и конечной точки через CLI
- [x] Отображение пути через лабиринт при его наличии в консоль.
- [x] Сохранение решения в файл.
