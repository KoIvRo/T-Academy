# О реализации
Игра запускается из src\main.py в интерактивном и не интерактивном режиме

Тесты запускаются из корневой папки python -m unittest discover -s tests pytest tests/

Протестировано на python 3.11 и 3.12
# Функциональные требования

## Не интерактивный (тестовый) режим работы

- [x] Программа должна иметь возможность запуска в не интерактивном (без взаимодействия с пользователем) режиме.
- [x] В не интерактивном режиме программа должна принимать два аргумента командной строки: "загаданное" слово и "угаданное".
- [x] Не интерактивный режим автоматически активируется при вызове программы с двумя аргументами командной строки.
- [x] От программы ожидается вывод на STDOUT в формате `угаданные_буквы_слова;результат`.
  Не угаданные буквы должны быть заменены символом звёздочки `*`.

## Интерактивный (пользовательский) режим работы

- [x] Интерактивный режим автоматически активируется при вызове программы без аргументов командной строки.
- [x] Программа должна выбрать случайную категорию (список) слов, если этого не сделал пользователь.
- [x] Программа должна выбрать случайный уровень сложности, если этого не сделал пользователь.
- [x] Программа должна выбирать случайное слово из выбранной категории (списка).
- [x] Игрок вводит одну букву за раз, чтобы угадать слово.
- [x] Экран должен обновляться после каждого ввода, показывая уже угаданные буквы и прочерки на местах неуказанных букв. (* на местах неуказанных букв, я поздно прочетал про прочерки)
- [x] Визуализация виселицы должна добавляться постепенно с каждой неправильной попыткой.
- [x] Игра завершается, когда слово угадано полностью или когда висельник полностью нарисован. (Игра продолжается с нарисованным висельником, если попыток больше, чем стадий виселицы)
- [x] Количество попыток ограничено и должно быть указано в начале игры.
- [x] Приводимая ниже информация относится к интерактивному режиму работы программы.
